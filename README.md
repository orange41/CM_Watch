## サイト概要
### サイトテーマ
業務上の失敗・事故情報の収集及び解決方法を現場から発信するサイト(Crisis Management Watch)
​
### テーマを選んだ理由
マニュアル通りにやったのはずなのに事故が起きた。見落としていた異常から事故につながった事例の共有が目的です。 
ハインリッヒの法則に基づき蓋然性が高く低コストの対応で大事故につながる事案の予防を目指します。
会社から渡されたマニュアルを補完および事故事例はより普段気づくようなうっかりを重点にしていきます。
現在働いている職場のポータルサイトの事故事例は重大事故が主題であるため、このようなサイトを共有したいと思いました。
ハインリッヒの法則によれば、1件の重大事故の背後には29件の軽微な事故、さらにその背後に300件の異常があるとされます
このため、多くの場合、まあいいやで済ます300件の異常こそが大事故を予防するために必要な問題であると考えます
このサイトはベテランよりもルーキーへ向けたサイトでありマニュアルにこう書かれているが間違ってしまった、ここからのリカバリーはどうすればいいのかというものを重視しています。
このため、ルーキー(あるいは経験豊富なベテラン)の小さなミスがなぜ起きたかを突き詰め、大事故を予防する目的でこのようなポートフォリオを制作してみようと考えました。

### ターゲットユーザ
・最近入社したあるいは初めて仕事をする人
​・ベテランだが復帰間もなく設備、資材の変遷に戸惑っている人

### 主な利用シーン
作業後に大事故にはつながらないが異常が発生した場合に使用。
自分でリカバリーできればその方法を入力し共有、わからない場合は解決方法がわかる人が追記する。
共有後にマニュアルを拡充させればなお良い。
